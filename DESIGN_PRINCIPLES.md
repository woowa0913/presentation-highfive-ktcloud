# 디자인 구현 원칙

이 문서는 HIGH FIVE 26 웹앱 프레젠테이션의 디자인 구현 시 반드시 지켜야 할 원칙을 정의한다.

## 핵심 원칙 1: 슬라이드 유형별 배경 분리

**간지(섹션 구분) 배경과 본문 배경은 반드시 분리한다.**

### 간지 페이지 (PPTX 슬라이드 3~6)

- 챕터(섹션) 전환 시에만 사용
- 각 챕터마다 고유한 배경 이미지 사용
- 내용: 섹션 번호(01, 02...) + 소제목만 배치
- 본문 콘텐츠를 간지 배경 위에 절대 배치하지 않는다

| 04 | 3_간지_D | image6.png | 진한 블루/핑크 도트 패턴 |

### 본문 페이지 (PPTX 슬라이드 7~16)

- 실제 콘텐츠(텍스트, 이미지, 데모)를 담는 페이지
- 배경: 흰색 또는 밝은 단색 (배경 이미지 없음)
- 상단에 타이틀, 하단에 출처 표기 가능
- 내용에 따라 아래 10가지 레이아웃 중 적합한 것을 선택

3. **출처 영역**: 하단 87% 위치, 작은 글씨

## 핵심 원칙 6: 배경 이미지 내장 요소와의 충돌 방지

배경 이미지(image1~10)에는 "HIGH FIVE 26" 로고, "INDEX", "Q&A", "THANK YOU" 등의 텍스트가 이미 내장되어 있다. 콘텐츠 텍스트가 이 내장 요소와 겹치지 않도록 반드시 PPTX 원본 좌표를 기준으로 배치한다.

### 좌표 규칙 (PPTX 원본 기준)

| 슬라이드 유형 | 내장 요소 | 콘텐츠 시작 위치 |
|-------------|----------|---------------|
| 타이틀 (image1) | "HIGH FIVE 26" 좌상단 + 로고 우하단 | 제목: x=16.3%, y=22.2% / 소속: x=16.3%, y=55.5% |
| 목차 (image2) | "HIGH FIVE 26" 좌상단 + "INDEX" 좌측 | 번호: x=39%, y=22.8% / 항목: x=44.3%, y=22.8% |
| 간지 (image3~6) | "HIGH FIVE 26" 좌상단 | 번호: x=5%, y=37% / 소제목: x=16.3%, y=37% |
| Q&A 표지 (image7) | "Q&A" 중앙 내장 | 추가 텍스트 불필요 |
| 마무리 (image10) | "THANK YOU" + 로고 내장 | 추가 텍스트 불필요 |

**규칙: 배경 이미지에 내장된 텍스트/로고 영역을 침범하지 않는다. 모든 콘텐츠는 PPTX 원본 좌표 기준으로 position: absolute 배치한다.**

## 핵심 원칙 7: CSS 기술 원칙 (Reveal.js 6.x)

Reveal.js 6에서 section 요소는 `position: absolute`로 관리된다.

1. **section의 position을 override하지 않는다** — `position: relative !important` 금지. Reveal.js의 슬라이드 전환이 깨진다.
2. **section에 `height: 100% !important`를 설정한다** — Reveal.js 6에서 section height가 0이 되어 자식의 `%` 기반 위치가 작동하지 않는 문제 해결.
3. **좌표 배치는 `position: absolute` + `top/left` %로 한다** — section height가 보장되면 % 기반 배치가 정상 작동.
4. **CSS 변경 시 `?v=N` 캐시 버스팅** — 브라우저 캐시로 인해 CSS 변경이 반영 안 되는 문제 방지.

## 핵심 원칙 8: 금지 사항

1. **간지 배경(image3~6)을 본문 콘텐츠 슬라이드에 사용하지 않는다**
2. **본문 레이아웃(흰 배경)을 간지로 사용하지 않는다**
3. **image7, image10처럼 텍스트가 내장된 배경 위에 중복 텍스트를 넣지 않는다**
4. **한 슬라이드에 본문 레이아웃을 혼합하지 않는다** (하나의 레이아웃만 선택)
5. **배경 이미지의 내장 로고/텍스트 영역과 콘텐츠 텍스트를 겹치지 않는다**
