# PRD: gemini 브랜치 특화 요구사항 및 구현 현황

본 문서는 `gemini` 브랜치에서 추가로 요청되고 구현된 핵심 기능과 설계 변경 사항을 정리합니다. 기존 `PRD.md`를 기반으로 하되, 바이브 코딩 실습 및 시각적 임팩트를 위한 특수 기능들이 포함되어 있습니다.

---

## 1. 시각적 경험 혁신 (Visual Vibe)

### 1.1 Hero 영상 배경 시스템
- **기능:** 슬라이드 레이어 뒤에 뷰포트 전체를 채우는 영상 배경(`assets/images/intro_video.mp4`) 배치.
- **구현:** `hero-fullscreen` 컨테이너 및 `section[data-chapter="hero"]` 감지 로직.
- **특수 효과:** 
    - **Pixel-level Color Mapping:** 영상의 특정 좌표 색상을 텍스트에 입히는 `background-clip: text` 효과 구현 (`js/hero-text-clip.js`).
    - **Dynamic Alignment:** 뷰포트 크기 변화에 맞춰 로고와 텍스트가 영상 내 특정 피사체와 정렬되도록 동적 계산 루틴 추가.

### 1.2 Interactive Beam Effect
- **기능:** 마우스 포인터를 따라다니는 Ribbon Trail(Knolskape 스타일) 효과.
- **구현:** `js/beam-effect.js`를 통한 Canvas 기반 애니메이션.
- **제어:** 상단 네비게이션 바에서 'Effect ON/OFF' 토글 기능 제공.

--- **상태:** ✅ 구현 완료 (배경 비디오 클립 기반 컬러링)

### 3. 고도화된 레이아웃 및 이론적 배경 (전문 컨퍼런스용 보강)
- **전이(Transfer)의 실패 (Ch 1):** 교육장 10%, 현업 70%의 전이 과정과 누수 요인 시각화.
- **HRD 로드맵 진화 (Ch 2):** Operator → Curator → Designer → Maker로 이어지는 역할을 박스형 그래픽으로 시각화.
- **HRD 메이커 5대 핵심 역량 (Ch 2):** Thinking, Context, Framework, Checkpoint, Debugging 능력을 레이더 차트로 시각화.
- **HRD DNA 매핑 (Ch 2):** 5대 핵심 역량을 기존 HRD의 문제 발굴, 설계, 피드백 등과 1:1로 매핑.
- **AI-Augmented ADDIE (Ch 2):** 전통적 ADDIE 모델의 각 단계별 AI 증강 요소(분석, 설계, 개발, 실행, 평가) 상세화.
- **도파민 루프(Dopamine Loop) (Ch 4):** 불편 발견 → 프롬프트 → 프로토타입 → 작은 성공 → 반복 개선의 사이클 시각화.
- **3단계 퀵스타트 고도화 (Ch 5):** Discover, Sync Vibe, Fast Iteration 체계로 구체화.
- **리스크와 극복 방안 (Ch 6):** 보안, 할루시네이션, 비용 리스크 및 폐쇄형 AI, 마스킹, 리터러시를 통한 해결 방안 추가.
- **상태:** ✅ 구현 완료

### 4. 챕터 기반 네비게이션 & 이펙트 토글
- **기능:** 전체 발표 흐름(Intro, Title, Index, 01~06, End)을 실시간으로 보여주는 상단 바.
- **세부 UI:**
    - 마우스 오버 시에만 나타나는 Auto-hide 로직.
    - 챕터 내 상세 페이지 이동을 위한 Page Dots 시스템.
    - 현재 위치 자동 활성화(Active State) 연동.

## 2. 사용자 내비게이션 및 UI

### 2.1 챕터별 상단 네비게이션 바
- **기능:** 전체 발표 흐름(Intro, Title, Index, 01~06, End)을 실시간으로 보여주는 상단 바.
- **세부 UI:**
    - 마우스 오버 시에만 나타나는 Auto-hide 로직.
    - 챕터 내 상세 페이지 이동을 위한 Page Dots 시스템.
    - 현재 위치 자동 활성화(Active State) 연동.

### 2.2 커스텀 컨트롤 버튼
- **기능:** 화면 중앙 좌측/우측에 배치된 대형 화살표 내비게이션.
- **지능형 색상 변경:** 슬라이드 배경이 어두운지(`data-background-color`) 또는 `.dark` 클래스인지 감지하여 버튼 색상을 흰색/검정색으로 자동 전환.

## 4. 기술적 개선 및 버그 수정

### 4.1 레이아웃 안정화
- **정렬 수정:** 모든 본문 슬라이드의 `.slide-body`가 `wanted-theme.css` 기준에 맞춰 일관되게 정렬되도록 조정.
- **이미지 최적화:** `pptx-media` 소스를 활용한 카드형 레이아웃(Crop_A/B/C) 전면 적용.

### 4.2 주요 버그 해결
- **HTML 구조 정상화:** 닫는 태그(`</div>`) 누락으로 인해 슬라이드 번호가 10번 이후 멈추던 현상 해결.
- **Reveal.js 정합성:** `window.Reveal` 전역 객체 노출을 통해 외부 스크립트(`demo-loader.js` 등)와의 연동성 확보.

## 5. 향후 관리 방향 (gemini 전담)
- **독립성 유지:** `main` 브랜치의 초기 정규 빌드 프로세스와 별개로, `gemini` 브랜치는 '바이브 코딩의 결과물'로서 가장 화려하고 인터랙티브한 최신 실험 상태를 유지함.
- **브랜치 라벨:** 화면 좌측 하단에 `gemini 브랜치` 전용 라벨을 상시 노출하여 현재 실행 환경 명시.
