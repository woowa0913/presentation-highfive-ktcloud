**발표자:** 원티드랩 김석준

## 1. 개요

원티드랩 HIGH FIVE 26 컨퍼런스에서 사용할 **웹 기반 프레젠테이션**을 제작한다.  
주관사가 제공한 PowerPoint 템플릿(HIGH FIVE 26_presentation.pptx)의 디자인을 충실히 웹으로 재현하되, PowerPoint에서는 불가능한 인터랙티브 요소를 추가한다.

### 발표 정보

| 항목 | 내용 |
|------|------|
| 타이틀 | 방법을 핏하게, 바이브 코딩으로 여는 'HRD 메이커' 시대 |
| 시간 | 45-60분 |
| 청중 | HRD 전문가 (비개발자) |
| 슬라이드 수 | 35-40장 (장당 1-1.5분) |

### 강연 소개글

> 이제 HRD는 교육 운영을 넘어 조직에 맞는 '방법'을 직접 설계하는 메이커가 되어야 합니다.
> 개발자나 외주 업체 없이도 AI와 바이브 코딩을 활용해 온보딩과 리더십 툴을 직접 제작한
> 혁신 사례를 소개합니다. 기술을 도구 삼아 교육의 효과성을 극대화하고 HRD의 존재 가치를
> 증명하는 메이커 시대의 생존 전략을 확인해 보세요.

### 메타 스토리 (핵심 설득 장치)

이 프레젠테이션 자체가 바이브 코딩으로 만들어졌다는 사실이 발표의 클라이맥스.  
"매체가 곧 메시지" — 내용과 형식이 일치하는 발표.

## 2. 디자인 요구사항

### 2.1 절대 원칙

- 주관사(원티드랩) 템플릿 디자인을 **최대한 충실히** 유지
- 배경, 레이아웃, 로고, 폰트 모두 원본 소스에서 정확히 반영
- **간지(섹션 구분) 배경과 본문 배경은 반드시 분리한다** (상세 규칙: DESIGN_PRINCIPLES.md 참조)
  - PPTX 슬라이드 3~6 배경(image3~6) → 챕터 간지 페이지에서만 사용

## 3. 기능 요구사항

### 3.1 필수 기능

| ID | 기능 | 설명 | 우선순위 |
|----|------|------|---------|
| F-01 | 슬라이드 네비게이션 | 화살표 키, 클리커로 슬라이드 전환 | P0 |
| F-02 | 애니메이션/트랜지션 | 슬라이드 전환 fade, 텍스트 순차 등장 (fragment) | P0 |

| F-05 | 발표자 모드 | 타이머, 발표 노트, 다음 슬라이드 미리보기 (S 키) | P0 |

| ID | 기능 | 설명 | 우선순위 |
|----|------|------|---------|
| I-01 | 클릭 인터랙션 | 슬라이드 내 요소 클릭 시 추가 정보 표시 등 | P1 |
| I-02 | 코드 실행 데모 | 바이브 코딩 과정을 보여주는 인터랙티브 요소 | P2 |
| I-03 | 라이브 데모 네비게이션 | iframe 내 데모에서 직접 클릭/조작 가능 | P1 |

### 3.3 iframe 데모 시스템 (demo-loader.js)

- `slidechanged` 이벤트로 현재 슬라이드 감지
- `data-src` → `src` 동적 설정 (lazy loading)
- 슬라이드 이탈 시 `src` 해제 (메모리 관리)
- 로딩 스피너 표시
- 5초 타임아웃 시 fallback 스크린샷/영상 전환
- X-Frame-Options 차단은 사전 `curl -I` 테스트로 확인
- 차단된 URL은 스크린 레코딩(mp4, 1920x1080, 최대 30MB) 또는 스크린샷(png) 사용

## 9. 성공 기준

1. PPTX 템플릿 디자인 충실 재현: 배경 이미지 100%, 색상 팔레트 CSS 변수 전수 매핑, 동일 폰트 패밀리/웨이트 사용
2. 주요 레이아웃 패턴 5종 이상 구현 (타이틀, 목차, 섹션 구분, 콘텐츠, 데모, Q&A, 마무리)
3. iframe 데모 안정 작동 (fallback 포함)
4. 애니메이션/트랜지션이 자연스럽고 비개발자 청중에게 "세련됨" 느낌
5. 발표자 모드(타이머, 노트) 정상 작동
6. 키보드/클리커 네비게이션 완벽 작동
7. 메타 리빌 슬라이드에서 청중 반응 유도
8. 마지막 페이지가 원본 PPTX와 동일하게 표시
