## 3. 현재 상태

- `final` 브랜치는 현재 `gemini` 브랜치 내용을 기준으로 유지한다.
- 현재 차이는 `final` 전용 기록 문서, 좌측 하단 `final 브랜치` 라벨, 그리고 본문 슬라이드의 Stitch 레퍼런스 기반 전면 재구조화다.
- `intro`, 타이틀, 간지, `end` 페이지는 기존 구조와 톤을 유지한다.
- 본문 슬라이드는 기존 내용을 부분 보정하는 방식이 아니라, 레퍼런스 톤을 기본 디자인 시스템으로 삼아 카드/브라우저/에디토리얼 레이아웃으로 다시 설계했다.
- 라이트/다크는 의미 단위로 묶어 적용하고, 상단 프로그레스바는 디자인만 바꾸되 hover 시에만 보이도록 유지한다.
- 본문 슬라이드의 `Chapter` 칩은 좌측 인라인 방식이 아니라 우측 상단 영역으로 옮겨, hover 시 나타나는 상단 UI와 같은 맥락의 정보 레이어로 정리한다.
- 본문 타이틀은 칩과 분리해 더 왼쪽에서 시작하도록 정렬하고, 의미가 있는 3개 슬라이드에는 지속 모션 이펙트를 제한적으로 적용한다.
- 9페이지는 `gemini` 브랜치 9페이지의 2카드 구조를 기준으로 다시 정리하고, 블루 톤안과 원래 톤 비교안을 함께 유지한다.
- 2페이지는 여러 장의 분리된 장표처럼 넘어가는 구성이 아니라, 한 장 안에서 텍스트가 순차 등장하고 이동하는 상태형 애니메이션으로 관리한다.
- 원래 톤 비교안 슬라이드는 삭제했고, 발견 슬라이드는 좌측 설명 + 우측 4단계 이미지 전환 구조로 유지한다.
- 발견 슬라이드의 마지막 단계에서는 `[ 방대한 양, 알 수 없는 기호, 복잡한 언어 ] / 매번 'Hello World'에서 멈추던 과거...` 캡션이 나타난다.
- 직접 시연 슬라이드는 현재 12페이지에 배치되어 있으며, 상단 `Tiny Case` 브리프와 하단 브라우저형 검색 결과 프레임을 함께 사용한다.
- `Chapter 02 · Why Maker` 구간의 16~20페이지는 `gemini`/`stitch` 레퍼런스를 기반으로 구조를 다시 정렬해, 구조적 한계-역할 진화-방법론 전환-ADDIE 재발견 흐름을 더 큰 타이포와 설명 밀도로 재구성했다.
- `Chapter 04 · Case Study` 간지 뒤에는 밸런스 게임 라이브용 슬라이드가 추가되었고, 실제 밸런스 게임 앱의 로컬 빌드를 발표 안에서 바로 실행할 수 있게 구성했다.
- `Chapter 02`는 현재 `왜 HRD는 메이커가 되어야 하는가`에 집중하도록 문제 인식과 전환 브릿지 중심의 5장 구조로 정리되어 있다.
- `Chapter 03`는 `HRD의 바이브 코딩, 개념이 아니라 실행 방식`이라는 이름으로, 체험-정의-방법론-역량-수렴의 7장 구조로 재배치되어 있다.
- 최신 커밋 정보와 배포 상태는 작업 종료 시점의 `final` HEAD 기준으로 관리한다.
- `main` 브랜치는 변경하지 않는다.

- 앞으로 모든 작업은 `final` 브랜치에서만 진행한다.

- 푸시 후 웹에서 볼 수 있는 링크를 함께 전달한다.
- `main` 브랜치는 건드리지 않는다.
