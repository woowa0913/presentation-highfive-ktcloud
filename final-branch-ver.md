## 3. 현재 상태

- `final` 브랜치는 현재 `gemini` 브랜치 내용을 기준으로 유지한다.
- 현재 차이는 `final` 전용 기록 문서, 좌측 하단 `final 브랜치` 라벨, 그리고 본문 슬라이드의 Stitch 레퍼런스 기반 전면 재구조화다.
- `intro`, 타이틀, 간지, `end` 페이지는 기존 구조와 톤을 유지한다.
- 본문 슬라이드는 기존 내용을 부분 보정하는 방식이 아니라, 레퍼런스 톤을 기본 디자인 시스템으로 삼아 카드/브라우저/에디토리얼 레이아웃으로 다시 설계했다.
- 라이트/다크는 의미 단위로 묶어 적용하고, 상단 프로그레스바는 디자인만 바꾸되 hover 시에만 보이도록 유지한다.
- 본문 슬라이드의 `Chapter` 칩은 좌측 인라인 방식이 아니라 우측 상단 영역으로 옮겨, hover 시 나타나는 상단 UI와 같은 맥락의 정보 레이어로 정리한다.
- 본문 타이틀은 칩과 분리해 더 왼쪽에서 시작하도록 정렬하고, 의미가 있는 3개 슬라이드에는 지속 모션 이펙트를 제한적으로 적용한다.
- 최신 커밋 정보와 배포 상태는 작업 종료 시점의 `final` HEAD 기준으로 관리한다.
- `main` 브랜치는 변경하지 않는다.

## 4. 변경 이력

### v1.0.0 (2026-04-14) - final 브랜치 초기화

- `gemini` 브랜치 최신 상태를 `final` 기준으로 채택
- `final-branch-ver.md`를 공식 기록 문서로 생성
- 좌측 하단 브랜치 라벨을 `final 브랜치`로 정리

### v1.1.0 (2026-04-14) - 본문 디자인 리스타일링

- Stitch 레퍼런스 코드의 톤을 반영해 본문 슬라이드 전용 디자인 토큰과 카드/글래스 패널 스타일 적용
- `intro`, 타이틀, 간지, `end` 페이지는 유지하고 `ch1`~`ch6` 본문만 리스타일링
- 상단 네비게이션과 프로그레스바를 hover 기반으로 정리하고, 라이트/다크 UI가 현재 슬라이드 톤에 맞게 바뀌도록 개선
- `도파민 루프` 슬라이드를 겹침 없는 루프 카드 레이아웃으로 재구성
- 사례 허브, KAC 단계 카드, 온보딩 카드, 밸런스 게임 선택지 등 잘림 위험이 큰 본문 카드형 레이아웃 보정

### v1.2.0 (2026-04-14) - 본문 전면 재구조화

- 사용자 요청에 따라 "기존 내용 위에 스타일 일부 적용" 방식이 아니라, Stitch 레퍼런스 톤을 기본 화면 언어로 삼아 본문 슬라이드를 전면 재설계
- `ch3`~`ch6`를 포함한 본문 전 구간을 에디토리얼 카드, 브라우저 목업, KPI 보드, 비교 프레임, 툴킷/거버넌스 레이아웃으로 다시 구성
- 외부 iframe 중심의 사례 표현을 발표용 화면 구조로 정리하고, 실제 자산 이미지 기반의 정적인 시연 구도로 교체
- 다크 슬라이드 안의 light card 대비 문제와 flex shrink로 인한 카드 축소 이슈를 디자인 시스템 레벨에서 보정
- expanded fragment 기준 대표 슬라이드를 로컬 브라우저 캡처로 확인해 잘림과 대비 문제를 점검

### v1.2.1 (2026-04-14) - 본문 타이틀 정렬 및 모션 포인트 보강

- 본문 전 슬라이드의 `Chapter` 칩을 우측 상단으로 이동하고, 상단 hover UI가 나타날 때는 자연스럽게 물러나도록 정리
- 본문 타이틀 영역을 왼쪽 기준으로 다시 정렬해, 예시 슬라이드인 `그래서 만들어보려고 했습니다`를 포함한 전체 본문 타이틀의 시작점을 통일
- 지속 모션은 `바이브 코딩은 개발자 흉내가 아니라 실행 방식이다`, `KAC 사례의 임팩트`, `실행의 동력이 되는 도파민 루프` 3개 슬라이드에만 제한 적용
- effect toggle과 연동해 새 모션도 함께 끌 수 있도록 처리하고, 로컬 캡처로 hover 상태와 본문 레이아웃을 재점검

## 5. 운영 규칙

- 앞으로 모든 작업은 `final` 브랜치에서만 진행한다.
- 작업이 끝날 때마다 이 문서에 변경 사항을 기록한다.
- 작업이 끝날 때마다 `final` 브랜치에 커밋하고 푸시한다.
- 푸시 후 웹에서 볼 수 있는 링크를 함께 전달한다.
- `main` 브랜치는 건드리지 않는다.
