# final 브랜치 버전 관리

이 문서는 `final` 브랜치의 기준 버전, 배포 링크, 변경 이력, 운영 규칙을 기록합니다.

## 1. 브랜치 기준

- 브랜치: `final`
- 복제 기준: `gemini` 브랜치 최신 상태
- 기준 커밋: `14ce5c7`
- 기준 커밋 메시지: `feat: add intro preamble slide 'Dadok time'`
- 목적: 발표 최종본 관리 및 승인본 추적

## 2. 웹 링크

- gemini 기준 원본 링크: `https://presentationvibe-git-gemini-woowa0913s-projects.vercel.app`
- final 브랜치 링크: `https://presentationvibe-git-final-woowa0913s-projects.vercel.app`
- preview 배포는 푸시마다 갱신되므로 기록 기준은 branch alias 링크로 유지한다.
- 마지막 확인 시각: 2026-04-14 16:48 KST
- Vercel 프로젝트: `presentation_vibe`

## 3. 현재 상태

- `final` 브랜치는 현재 `gemini` 브랜치 내용을 기준으로 유지한다.
- 현재 차이는 `final` 전용 기록 문서, 좌측 하단 `final 브랜치` 라벨, 그리고 본문 슬라이드의 Stitch 레퍼런스 기반 전면 재구조화다.
- `intro`, 타이틀, 간지, `end` 페이지는 기존 구조와 톤을 유지한다.
- 본문 슬라이드는 기존 내용을 부분 보정하는 방식이 아니라, 레퍼런스 톤을 기본 디자인 시스템으로 삼아 카드/브라우저/에디토리얼 레이아웃으로 다시 설계했다.
- 라이트/다크는 의미 단위로 묶어 적용하고, 상단 프로그레스바는 디자인만 바꾸되 hover 시에만 보이도록 유지한다.
- 본문 슬라이드의 `Chapter` 칩은 좌측 인라인 방식이 아니라 우측 상단 영역으로 옮겨, hover 시 나타나는 상단 UI와 같은 맥락의 정보 레이어로 정리한다.
- 본문 타이틀은 칩과 분리해 더 왼쪽에서 시작하도록 정렬하고, 의미가 있는 3개 슬라이드에는 지속 모션 이펙트를 제한적으로 적용한다.
- 9페이지는 `gemini` 브랜치 9페이지의 2카드 구조를 기준으로 다시 정리하고, 블루 톤안과 원래 톤 비교안을 함께 유지한다.
- 2페이지는 여러 장의 분리된 장표처럼 넘어가는 구성이 아니라, 한 장 안에서 텍스트가 순차 등장하고 이동하는 상태형 애니메이션으로 관리한다.
- 원래 톤 비교안 슬라이드는 삭제했고, 발견 슬라이드는 좌측 설명 + 우측 4단계 이미지 전환 구조로 유지한다.
- 발견 슬라이드의 마지막 단계에서는 `[ 방대한 양, 알 수 없는 기호, 복잡한 언어 ] / 매번 'Hello World'에서 멈추던 과거...` 캡션이 나타난다.
- 직접 시연 슬라이드는 현재 12페이지에 배치되어 있으며, 상단 `Tiny Case` 브리프와 하단 브라우저형 검색 결과 프레임을 함께 사용한다.
- 최신 커밋 정보와 배포 상태는 작업 종료 시점의 `final` HEAD 기준으로 관리한다.
- `main` 브랜치는 변경하지 않는다.

## 4. 변경 이력

### v1.0.0 (2026-04-14) - final 브랜치 초기화

- `gemini` 브랜치 최신 상태를 `final` 기준으로 채택
- `final-branch-ver.md`를 공식 기록 문서로 생성
- 좌측 하단 브랜치 라벨을 `final 브랜치`로 정리

### v1.1.0 (2026-04-14) - 본문 디자인 리스타일링

- Stitch 레퍼런스 코드의 톤을 반영해 본문 슬라이드 전용 디자인 토큰과 카드/글래스 패널 스타일 적용
- `intro`, 타이틀, 간지, `end` 페이지는 유지하고 `ch1`~`ch6` 본문만 리스타일링
- 상단 네비게이션과 프로그레스바를 hover 기반으로 정리하고, 라이트/다크 UI가 현재 슬라이드 톤에 맞게 바뀌도록 개선
- `도파민 루프` 슬라이드를 겹침 없는 루프 카드 레이아웃으로 재구성
- 사례 허브, KAC 단계 카드, 온보딩 카드, 밸런스 게임 선택지 등 잘림 위험이 큰 본문 카드형 레이아웃 보정

### v1.2.0 (2026-04-14) - 본문 전면 재구조화

- 사용자 요청에 따라 "기존 내용 위에 스타일 일부 적용" 방식이 아니라, Stitch 레퍼런스 톤을 기본 화면 언어로 삼아 본문 슬라이드를 전면 재설계
- `ch3`~`ch6`를 포함한 본문 전 구간을 에디토리얼 카드, 브라우저 목업, KPI 보드, 비교 프레임, 툴킷/거버넌스 레이아웃으로 다시 구성
- 외부 iframe 중심의 사례 표현을 발표용 화면 구조로 정리하고, 실제 자산 이미지 기반의 정적인 시연 구도로 교체
- 다크 슬라이드 안의 light card 대비 문제와 flex shrink로 인한 카드 축소 이슈를 디자인 시스템 레벨에서 보정
- expanded fragment 기준 대표 슬라이드를 로컬 브라우저 캡처로 확인해 잘림과 대비 문제를 점검

### v1.2.1 (2026-04-14) - 본문 타이틀 정렬 및 모션 포인트 보강

- 본문 전 슬라이드의 `Chapter` 칩을 우측 상단으로 이동하고, 상단 hover UI가 나타날 때는 자연스럽게 물러나도록 정리
- 본문 타이틀 영역을 왼쪽 기준으로 다시 정렬해, 예시 슬라이드인 `그래서 만들어보려고 했습니다`를 포함한 전체 본문 타이틀의 시작점을 통일
- 지속 모션은 `바이브 코딩은 개발자 흉내가 아니라 실행 방식이다`, `KAC 사례의 임팩트`, `실행의 동력이 되는 도파민 루프` 3개 슬라이드에만 제한 적용
- effect toggle과 연동해 새 모션도 함께 끌 수 있도록 처리하고, 로컬 캡처로 hover 상태와 본문 레이아웃을 재점검

### v1.2.2 (2026-04-14) - p2 흐름 재구성 및 6페이지 gemini 기준 정렬

- p2를 4단계 상태 전환형 레이아웃으로 재구성해 `다음` 클릭 시 첨부 시안 흐름대로 장면이 바뀌도록 구성
- `다문`에 `多聞`을 반영하고, 마지막 단계에서는 상단 문장과 핵심 키워드 강조색을 적용
- 6페이지 `교육의 효과성을 높이고 싶었습니다`를 `gemini` 브랜치 6페이지와 같은 미니멀 다크 슬라이드 흐름으로 재구성
- p2 단계별 캡처와 6페이지 fragment 단계별 캡처로 시각 비교를 수행

### v1.2.3 (2026-04-14) - 8페이지 카드 구조 정렬

- 8페이지 `그래서 만들어보려고 했습니다`의 3개 카드를 공통 구조로 재정렬
- 각 카드를 `아이콘 - 제목 - 3줄 설명 - 하단 요약 배지` 순서로 통일해 설명 영역 높이와 여백 리듬을 맞춤
- 슬라이드 하단에 `어느 경로든 시간, 비용, 통제권 — 셋 중 하나는 포기해야 하는 경우가 많다` 문장을 별도 바닥 패널로 추가
- 로컬 캡처로 8페이지 전체 표시 상태와 overflow 0을 확인

### v1.2.4 (2026-04-14) - 9페이지 장벽 슬라이드 비교안 추가

- 9페이지 제목 문구를 `우리를`에서 `나를`로 조정
- `gemini` 브랜치 9페이지의 핵심 정보 구조를 기준으로, 2카드 중심의 미니멀 레이아웃으로 재구성
- 첫 번째 안은 블루 포인트 톤으로, 두 번째 안은 원래 웜 톤으로 구성해 바로 비교할 수 있도록 연속 슬라이드 2장으로 배치
- 하단 메시지도 `나를 두렵게 했던 것은 논리가 아니라 낯선 도구였습니다` 흐름으로 정리

### v1.3.0 (2026-04-14) - p2 단일 애니메이션화 및 사례 구간 재정렬

- 2페이지를 분리 장면형이 아니라 단일 슬라이드 내부 상태 전환형 애니메이션으로 다시 구성하고, 양 옆 배경 톤이 끊겨 보이던 문제를 viewport 단까지 확장해 정리
- 6페이지 다크 슬라이드도 같은 방식으로 viewport 배경을 동기화해 좌우 검은 띠와 본문 배경 차이가 나지 않도록 수정
- 8페이지 카드의 이모지 박스 장식을 제거하고, 요약 구조는 유지한 채 아이콘을 더 직접적으로 보이게 조정
- 본문 공통 폰트를 `WantedSans` 우선으로 재정렬하고, 9페이지 장벽 슬라이드 박스는 더 좁고 위쪽에 배치되도록 보정
- 원래 톤 비교안을 삭제하고, 발견 슬라이드는 `gemini` 10페이지를 참고한 좌측 설명 + 우측 4단계 이미지 전환 구조로 재편
- 직접 시연 슬라이드는 12페이지로 재배치하고, `Build Spec` / `What Changed`를 제거한 뒤 가로형 `Tiny Case` 브리프와 브라우저형 검색 결과 시연 화면으로 다시 구성

### v1.3.1 (2026-04-14) - 12페이지 검색 스냅샷 및 앱 토글 추가

- 12페이지 브라우저 프레임 하단을 실제 검색 결과 스냅샷처럼 보이는 Google 이미지 검색형 레이아웃으로 재구성
- 상단 주소창 우측에 `검색 화면` / `앱 실행` 버튼을 추가해 같은 프레임 안에서 상태를 전환할 수 있도록 구현
- `앱 실행` 클릭 시 `https://mac-shortcut-master-v4.vercel.app/` 를 iframe으로 로드해 레이아웃 안에서 바로 시연 가능하도록 연결
- 로컬 브라우저에서 검색 스냅샷 상태와 앱 실행 상태를 각각 캡처해 표시 및 전환 동작을 검증

### v1.3.2 (2026-04-14) - GEN 발표형 전환점/도파민/현장 장면 보정

- 11페이지 `작은 성공이 만든 전환점`을 `gemini` 발표 스타일에 맞춘 GEN형 풀카피 슬라이드로 재구성하고, `직접 만들 수 있다는 자신감`과 핵심 확신 문장은 기존 블루 포인트를 유지
- 11페이지의 `도파민이 터졌습니다.` 문장은 노란 강조 대신 옅은 회색 보조 문장으로 정리해 전체 리듬을 더 차분하게 맞춤
- 13페이지 하단 선언문은 배경 패널을 제거하고, 한 줄 블루 텍스트로 확대해 가독성을 높인 뒤 문구를 `"만들었다"는 작은 도파민이 자기주도적 문제해결을 이끕니다. 그렇다면 HRD 담당자가 왜 직접 이 방식으로 만들어야 하는가?` 로 교체
- 13페이지 마지막 장면에는 슬라이드 전역에 시차를 두고 반짝임이 터지는 sparkle 레이어를 추가하고, `effects-off` 토글과도 연동
- 15페이지 `현장에서 반복되는 장면`은 `gemini` 브랜치 15페이지의 GEN식 내러티브 구조를 참고해 대형 중앙 카피와 블루 강조 박스로 재구성

## 5. 운영 규칙

- 앞으로 모든 작업은 `final` 브랜치에서만 진행한다.
- 작업이 끝날 때마다 이 문서에 변경 사항을 기록한다.
- 작업이 끝날 때마다 `final` 브랜치에 커밋하고 푸시한다.
- 푸시 후 웹에서 볼 수 있는 링크를 함께 전달한다.
- `main` 브랜치는 건드리지 않는다.
